---
name: generate_outline
description: 根据用户提供的主题、目标受众和写作目的，生成结构清晰、层级合理的文章大纲。此为技能，非工具函数，通过view_text_file工具查看技能内容
---

# When to Use
在以下场景触发该技能：
- 用户需要从零开始组织一篇文章 / 报告 / 视频脚本
- 用户已有模糊想法，需要结构化整理
- 用户希望优化已有内容结构

# Inputs
- 主题 / Topic（必需）
- 写作目标 / Goal（必需）（如解释、叙事、说服）
- 目标受众 / Audience（可选）
- 风格 / Style（可选）（如专业、故事化、口语化）
- 篇幅 / Length（可选）（如短文 / 长文 / 视频脚本）

# Outputs
- 分层结构的大纲（至少 2–3 层）
- 每个一级 / 二级标题具备明确语义
- 可选：每一节附带简要说明（1–2 行）

# Requirements / Success Criteria
一个高质量输出应满足：
- 结构完整：包含开头、主体、结尾
- 逻辑清晰：章节之间有递进或因果关系
- 层级合理：避免过深或过浅结构
- 贴合目标：围绕用户目标展开，而非泛化
- 可执行性强：可以直接用于写作

# Constraints / Guardrails
- 不生成无关内容或跑题结构
- 不使用空洞标题（如“介绍”“总结”需具体化）
- 避免重复或语义重叠的章节
- 若信息不足，应做合理假设并保持通用性

# Failure Modes（常见问题）
- 结构松散，没有清晰主线
- 标题过于抽象或模板化
- 各部分内容重复
- 未体现用户目标（如缺乏说服结构或故事节奏）
